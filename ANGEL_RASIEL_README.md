## 🔮 魔法攻撃リスト

Angel Rasielは以下の魔法を**ランダムな順序**で使用します：

| 魔法 | レベル | 使用回数 | 効果 |
|------|--------|----------|------|
| Divine Smite | 10 | 7 | 神聖な一撃 |
| Guiding Bolt | 10 | 15 | 誘導する光の矢 |
| Sunbeam | 10 | 12 | 太陽の光線 |
| Heal | 10 | 3 | 自己回復 |
| Lightning Lance | 10 | 7 | 雷の槍 |
| Flaming Barrage | 10 | 15 | 炎の弾幕 |
| Root | 10 | 1 | 拘束 |
| Magma Bomb | 10 | 15 | マグマ爆弾 |
| Thunderstorm | 10 | 1 | 雷雨召喚※ |
| Ball Lightning | 10 | 15 | 球状の雷 |

※ Thunderstormは効果中は再詠唱しません

**合計**: 91回の魔法攻撃 + 無敵時間無視の近接攻撃

### 🎲 ランダム順序の仕組み

- 戦闘開始時に91個の魔法がランダムにシャッフルされます
- 毎回異なる順序で魔法を使用するため、パターンが読めません
- 全ての魔法を使い終わったら、再度シャッフルして繰り返します

## ⚙️ カスタマイズ方法

### 体力や攻撃力の変更
`AngelRasielEntity.java` の `createAttributes()` メソッドを編集：
```java
.add(Attributes.MAX_HEALTH, 300.0D)      // 体力
.add(Attributes.ATTACK_DAMAGE, 15.0D)    // 攻撃力
.add(Attributes.ARMOR, 15.0D)            // 防御力
```

### 魔法の種類や回数の変更
`AngelRasielAttackGoal.java` の `initializeRandomSpellSequence()` メソッドを編集：
```java
// Divine Smite x7 → 回数を変更
addSpellsToList(allSpells, SpellRegistry.DIVINE_SMITE_SPELL.get(), level, 7);

// 新しい魔法を追加
addSpellsToList(allSpells, SpellRegistry.YOUR_SPELL.get(), level, 回数);
```

### 魔法のクールダウン変更
`AngelRasielAttackGoal.java` の `resetSpellAttackTimer()` メソッドを編集：
```java
this.spellAttackDelay = 2; // 魔法間のクールダウン（ティック数）
// 2 = 0.1秒、20 = 1秒
```

### ボスバーの色を変更
`AngelRasielEntity.java` のコンストラクタを編集：
```java
BossEvent.BossBarColor.YELLOW  // 金色
// 他の色: PINK, BLUE, RED, GREEN, PURPLE, WHITE
```

### 無敵時間無視を解除
`AngelRasielEntity.java` の `doHurtTarget()` メソッドをコメントアウトまたは削除

## 🎯 戦闘のヒント

### プレイヤー向け
1. **パターンが読めない**: 魔法の順序がランダムなので要注意
2. **無敵時間が効かない**: 近接攻撃を受けると連続ダメージ
3. **距離を保つ**: 魔法と剣の両方に注意
4. **回復に注意**: Healで3回回復する
5. **ボスバーで体力確認**: 画面上部で残り体力を確認できる
6. **ノックバック無効**: 吹き飛ばし攻撃が効かない

### 戦闘の難しさ
このAngel Rasielは非常に強力です：
- ✅ ランダムな魔法順序
- ✅ 無敵時間無視の攻撃
- ✅ 高い体力と防御力
- ✅ 91回の魔法攻撃
- ✅ ノックバック無効

### 難易度調整したい場合
難しすぎる場合：
- 体力を減らす（300 → 150など）
- 魔法の回数を減らす
- 無敵時間無視を解除
- 攻撃力を下げる
- ノックバック耐性を下げる

## 🐛 トラブルシューティング

### ボスバーが表示されない
- サーバー側のコードが正しく実装されているか確認
- プレイヤーが範囲内にいるか確認

### 魔法の順序が同じ
- 完全にランダムなので、稀に似た順序になることがある
- 再度召喚すれば異なる順序になる

### 無敵時間が無視されない
- `doHurtTarget()` メソッドが正しく実装されているか確認
- エンティティがプレイヤーを攻撃しているか確認

### 近接攻撃をしない
- プレイヤーに十分近づいているか確認
- ターゲットが設定されているか確認

## 📊 技術的な詳細

### 無敵時間無視の実装
```java
@Override
public boolean doHurtTarget(Entity target) {
    if (target instanceof LivingEntity livingTarget) {
        livingTarget.invulnerableTime = 0;  // 攻撃前に無敵時間を0に
        // 攻撃処理
        livingTarget.invulnerableTime = 0;  // 攻撃後も無敵時間を0に
    }
}
```
